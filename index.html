In my Team Software class here at Michigan Tech, we're finishing up Sorting Hat, a tool that sorts teams after teammate preferences have been gathered via survey. The web app sorts the students into teams using a genetic algorithm optimized on the preferences of the students. I wanted to add a feature that said "Out of 1,345,500 possible arrangements, we've found the following:". The following details how I calculated this number. The problem: Given x people in the population, how many different ways can you split them into y teams? The teams are as close in size as possible. The first step I took was to create an array describing how many people are on each team. This is simple enough, just distribute the players evenly until you are out of players. 
[sourcecode language="csharp"]

int[] teamSizes = new int[numTeams];

            int pop = popSize;

            int idx = 0;

            while (pop > 0)

            {

                teamSizes[idx++ % numTeams] += 1;

                pop -= 1;

            }
